Scheme-part1

Scheme是一个著名的函数式编程语言，它是Lisp的一种形式。

在学习Sheme的过程中，我们将对解释器即interpreter进行了解，并研究解释器的设计和他们在执行程序时创建的计算过程。程序语言只是文本的集合，只有通过解释器我们才能执行对应的计算。

一般解释器都有一个共同的结构：两个相互递归的函数。

  • 一个评估环境中的表达式
  • 一个将函数应用于参数

因为是相互递归的，所以在应用其中一个函数时需要应用更多的函数。

Ants

此游戏是模仿植物大战僵尸的游戏，用蚂蚁和蜜蜂来代替植物与僵尸，大致玩法类似，相信大家都玩过。

Core concepts

The colony:
This is where the game takes place. The colony consists of several Places that are chained together to form a tunnel where bees can travel through. The colony also has some quantity of food which can be expended in order to place an ant in a tunnel.

Places:
A place links to another place to form a tunnel.The player can put a single ant into each place. However, there can be many bees in a single place.

The Hive:
 This is the place where bees originate. Bees exit the beehive to enter the ant colony.

This part is a summary of Lists, Data Abstraction, Trees.

Lists

A list is a data structure that can store multiple elements. Each element can be any type, even a list itself. We write a list as a comma-separated list of expressions in square brackets:

Each element in the list has an index, and you should notice that the first element in the list whose index is 0 but not 1.

For list[a: b], this is like as [a, b).

Data Abstraction

Data abstraction is a powerful concept in computer science that allows programmers to treat code as objects. For example, using code to represent cars, chairs, people, and so on. That way, programmers don’t have to worry about how code is implemented; they just have to know what it does.

What is the most important is: Constructors and Selectors

  • Constructors: functions that build the abstract data type
  • Selectors: functions that retrieve information from the data type

Trees

A Tree is a data structure that represents a hierarchy of information.

A good simple is the file system:

As you can see, unlike trees in nature, the tree abstract data type is drawn with the root at the top and the leaves at the bottom.

Here are some tree terminology:

  • root: the node at the top of the tree
  • label: the value in a node
  • branches: a list of trees directly under the tree’s root
  • leaf: a tree with zero branches
  • node: any location within the tree

Some general functions:

  • tree(label, [branches = [])
  • label(tree)
  • branches(tree)
  • is_tree
  • is_leaf

cs61a_Cats

Cats

Cats 项目的目标是开发一个打字测速器。

这里是项目完成后的一个样品: Cats

我们将此项目分成三部分来逐渐完成。你需要在这过程中陆续解决10个问题。

Prat 1: Typing

本部分主要是解决打字的速度和准确度问题。

Problem 1

实现choose函数，其中包含以下三个参数：

  • a list of paragraphs
  • a select function, which returns Ture for paragraphs that can be selected
  • a non-negative index k

函数choose将会返回第k个符合函数select的词组。

如果没有这样的自然段存在（因为k太大了），函数返回空字符串。

Problem 2

about是返回值为函数的一个函数。返回值函数可以用于choose函数，方便以后调用。

此函数有一个参数topic，topic是一个字符串组，返回值函数是来判断词组中是否含有topic中的单词。

Tips: 不分大小写和标点符号，且要学会使用utils.py函数。

Problem3

实现accuracy函数，此函数引用了两个参数，typed和reference，返回值为typed与reference的匹配度。

需要注意的是，当两者均为空时，匹配度为100，而一方为0时，匹配度为0。

Problem4

wpm函数，全称为word per minute, 是用来计算打字速度的函数，其中，他不是用的每个单词并不是指一个实际的单词，而是用5个字母来代替一个单词

至此，这个项目已经能简单的运行并测试你的打字速度了

Part 2

本部分主要是解决自动修正问题。

Problem 5

实现一个函数autocorrect， 这个函数有四个参数，分别是typed_words, valid_words, diff_function and limit。

  • 如果typed_words是包含在valid_words里面，那么此函数直接返回这个单词。
  • 如果存在差异，那么用diff_function来判断。

diff_function返回值为一个数字，一般情况是返回差异最小的单词，但是当diff_function的返回值大于limit时，返回typed_word。

Hints: 尝试使用max 和min函数，已经可选择的key参数

Problem 6

实现函数feline_flips，这个函数可用于problem 5中的diff_function函数。

先看需要替换的字母数，再加上样本和标准之间的长度差。

使用递归

Problem 7

实现minimum_mewtation函数，这个函数返回从start到goal所需变化的最小步数。

这里有三种情况：

  • 增加一个字母
  • 删除一个字母
  • 替换一个字母

每个变化记为1.

本函数已经提供了模板，是一个递归结构，里面包含三个递归函数，其中有一个递归函数与problem 6 中的feline_flips相似。

至此，现在的项目已经可以利用自动更正系统辅助打字了

Part 3

本部分主要是解决多人打字问题。

Problem 8

实现report_progress函数，参数为sofar, prompt, user_id和upload。其中，prompt中的单词不少于sofar。

如何计算进度？在第一个错误词汇前拼写正确的单词数除以prompt的总单词数即可。

Problem 9

实现time_per_word函数，它的参数是words和times_per_player，都是列表。返回值为match。

match是一个抽象数据，里面包含words和times。其中，times是一个双重列表。times[i][j]表示第i个玩家拼写第j个单词的时间。

这个问题主要是检测对于抽象数据的实现，而不是像以前的列表和字典一样，仅仅只是特定的数据结构。

For example: 如果times_per_player = [[1, 3, 5], [2, 5, 6]], 那么对应的match应该是[[2,2], [3, 1]]。因为是由(3-1)，(5-3)得来的。

Problem 10

实现fastest_words函数，此函数返回玩家打字最快的单词。参数为match。

需要注意的是，返回值是一个列表的列表，而且当两个玩家打字速度相同时，认为这个单词最快速度属于前一个玩家。

match是problem 9中的返回值，一个抽象数据。要学会使用word_at函数，参数是match和word_index。
