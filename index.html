基本概念

数据：数据是数据库中存储的基本对象，分为数值数据和非数值数据；

特点：数据和语义是不可分的

数据库：数据库是长期储存在计算机内、有组织的、可共享的大量数据的集合

基本特征：

  • 数据按一定的数据模型组织、描述和储存
  • 可为各种用户共享
  • 数据间联系密切，具有最小冗余度和较高的独立性
  • 服务与某个特定的应用

数据库模式：数据库中全体数据的逻辑结构和特征的描述

数据—————>数据库

数据的语义——>数据库模式

数据库管理系统（DBMS）：计算机程序的集合，用于创建和维护数据库

  • 位于操作系统和用户应用之间
  • 总是基于某种数据模型
  • 数据厂商的产品通常是指DBMS

数据库系统（DBS）：在计算机系统中 引入了数据库后的系统，即采用了数据库技术的计算机系统

SQL的基本组成

  • DDL：定义基本表、视图操作、索引操作
  • DML：数据记录操作
  • DCL：访问控制

  • Procedure: 对于scheme，procedure是一种函数，而在python中，procedure是一个实例，可以根据此实例来执行
  • special forms: 除了初始函数与值外，我们应当能够应用一些自定义函数和特殊函数，这也是抽象的一部分

  • Symbol evaluation
  • Calling built-in procedures

Frame

首先是框架（环境）：Frame类，每个frame创建时继承上一个frame，Global Frame的父框架是None。

一个框架至少有两个实例属性：

  • bindings: 一个字典类型，它联系了Scheme Symbol和Scheme value。
  • parent: 父框架。

而一个框架同时也有以下方法：

  • define: 在一个frame实例中链接symbol和value
  • lookup: 在一个框架下寻找symbol对于的value，如果没找到，那么在它的父框架中再寻找，如果始终无法找到，那么返回SchemeError。

Interpreters

计算器语言通过嵌套调用表达式提供了一种组合方式。但是，计算器语言没有办法定义新的运算符，绑定名字与值，或者表达一般的计算式。计算器不支持抽象。因此，它不是一个强有力的编程语言。

我们现在应要去定义一种常规的编程语言，可以绑定名字与值和定义新运算来支持抽象。

与上一节将完整的解释器呈现为Python源代码不同，本节采用描述性方法。配套项目要求您通过构建一个功能齐全的Scheme解释器来实现这里提出的想法。

Eval/apply recursion:

实现评估过程的函数 scheme_eval和scheme_apply是相互递归的。每当遇到调用表达式时，评估都需要应用。应用程序使用评估将操作数表达式评估为参数，以及评估用户定义过程的主体。这种相互递归过程的一般结构相当普遍地出现在解释器中：评估是根据应用定义的，应用是根据评估定义的。

这个递归循环以语言初始值结束。评估有一个评估原始表达式的基本情况。一些特殊形式也构成没有递归调用的基本情况。函数应用程序有一个应用原始过程的基本案例。这种相互递归的结构，在处理表达式形式的eval函数和处理函数及其参数的apply函数之间，构成了评估过程的本质。

2. Environments

现在我们已经描述了Scheme解释器的结构，现在要去实现Frame类来形成环境，每个France实例代表了一种环境，其中符号绑定值。

一个frame有一个bindings字典，并且有一个父框架，而对于全局框架来说，父框架就是None。

绑定不是直接访问的，而是通过两个方法：lookup和define。第一个方法是实现了第 1 章中描述的计算环境模型的查找过程。一个符号和当前框架下的bindings匹配。如果找到了，那么返回对应值。如果没找到，那么lookup会查找父框架。另一方面，define方法总是在当前框架下绑定符号与值。

以类似的方式，我们可以将 Scheme 解释器视为一个非常特殊的机器，它将机器的描述作为输入。给定这个输入，解释器将自己配置为模拟所描述的机器。例如，如果我们为我们的评估器提供阶乘的定义，那么评估器将能够计算阶乘。

从这个角度来看，我们的 Scheme 解释器被视为一台通用机器。当这些机器被描述为 Scheme 程序时，它会模仿其他机器。它充当由我们的编程语言操作的数据对象和编程语言本身之间的桥梁。用户在我们正在运行的 Scheme 解释器中键入一个 Scheme 表达式的图像。从用户的角度来看，诸如(+ 2 2)之类的输入表达式是编程语言中的表达式，解释器应该对其求值。然而，从 Scheme 解释器的角度来看，表达式只是一个单词的句子，要根据一组明确定义的规则进行操作。

用户的程序是解释器的数据不必成为混淆的来源。事实上，有时忽略这种区别并让用户能够将数据对象显式评估为表达式是很方便的。在 Scheme 中，无论何时使用运行过程，我们都会使用此工具。Python 中也存在类似的函数：eval函数将计算 Python 表达式，exec函数将执行 Python 语句。因此，

实现report_progress函数，参数为sofar, prompt, user_id和upload。其中，prompt中的单词不少于sofar。

如何计算进度？在第一个错误词汇前拼写正确的单词数除以prompt的总单词数即可。

Problem 9

实现time_per_word函数，它的参数是words和times_per_player，都是列表。返回值为match。

match是一个抽象数据，里面包含words和times。其中，times是一个双重列表。times[i][j]表示第i个玩家拼写第j个单词的时间。

这个问题主要是检测对于抽象数据的实现，而不是像以前的列表和字典一样，仅仅只是特定的数据结构。

For example: 如果times_per_player = [[1, 3, 5], [2, 5, 6]], 那么对应的match应该是[[2,2], [3, 1]]。因为是由(3-1)，(5-3)得来的。

Problem 10

实现fastest_words函数，此函数返回玩家打字最快的单词。参数为match。

需要注意的是，返回值是一个列表的列表，而且当两个玩家打字速度相同时，认为这个单词最快速度属于前一个玩家。

match是problem 9中的返回值，一个抽象数据。要学会使用word_at函数，参数是match和word_index。
